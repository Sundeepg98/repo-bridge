How device flow works

Device flow is GitHub's no-password way to authorize a device that can't safely hold a secret: you approve access by typing a short code at github.com/login/device, and GitHub hands the sandbox a scoped, short-lived token — no password shared, nothing to paste back.

01

Request

A sandbox requests access and is handed a short code to show you.

02

Approve

You enter it once at github.com/login/device and confirm. Your password never leaves GitHub.

03

In sync

GitHub issues a scoped token. The sandbox can now clone and push the repositories you selected — nothing else.

Assistant won't fetch links? Here's the same connection, spelled out step by step.
full prompt 
Connect to my private GitHub repository <owner/repo> through a GitHub App using OAuth device flow.

Client ID: YOUR_CLIENT_ID

Steps:
1. Request a device code and user code from GitHub's device-flow endpoint for that Client ID.
2. Show me the user code and tell me to enter it at https://github.com/login/device, then wait while I authorize.
3. Exchange the authorization for a scoped, short-lived user-access token.
4. Confirm you can clone <owner/repo>, commit, and push back — and that access is limited to that repository only.

Never print the token.

What it can touch

Reach
Only the repositories you select
Access
Read + write to code; metadata read-only
Direction
Two-way — clone down, push back
Credentials
None exposed — no password, no PAT
Token
Issued per login, expires in 8h
Revoke
Token: Authorized GitHub Apps. App access until you uninstall.

The ~8h token is only the sandbox's working credential. Granting your repos also installs the app on them — a separate, lasting grant: while it stands, the app's owner can reach those repos at any time, with or without that token. Revoking the token doesn't remove it. To fully disconnect, do both above — revoke the token and uninstall the app.

Not granted: other repos · Actions/Codespaces secrets · org settings · repo deletion
