GitHub App · @Sundeepg98

sundeepg98-repo-bridge

Keeps the repositories you own and the AI sandboxes that work on them in sync — clone down, push back.

Configure repositories → How device flow works ↗

How it works

01

Request

A sandbox requests access and is handed a short code to show you.

02

Approve

You enter it once at github.com/login/device and confirm. Your password never leaves GitHub.

03

In sync

GitHub issues a scoped token. The sandbox can now clone and push the repositories you selected — nothing else.

What it can touch

Reach
Only the repositories you select
Access
Read + write to code; metadata read-only
Direction
Two-way — clone down, push back
Credentials
None exposed — no password, no PAT
Token
Issued per login, short-lived
Revoke
Anytime, from GitHub settings

App details

Public identifiers, safe to share. The Client ID is what a sandbox uses to begin the device-flow login — no secret is ever shown here.
