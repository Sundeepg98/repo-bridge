Device-flow GitHub bridge

repo-bridge

Keeps the repositories you own and the AI sandboxes that work on them in sync — clone down, push back.

How it works

01

Request

A sandbox requests access and is handed a short code to show you.

02

Approve

You enter it once at github.com/login/device and confirm. Your password never leaves GitHub.

03

In sync

GitHub issues a scoped token. The sandbox can now clone and push the repositories you selected — nothing else.

Connect a sandbox

Drop one line into a chat you're already in with an internet-capable AI assistant. It fetches the full device-flow steps itself and runs them — no new tab, nothing to assemble by hand. Just name your repo.

Connect my private GitHub repo <owner/repo> to this sandbox via the repo-bridge GitHub App (Client ID YOUR_CLIENT_ID) — fetch https://sundeepg98.github.io/repo-bridge/connect.md and follow its device-flow steps. Don't print the token.

If your assistant can fetch that link, its sandbox can almost certainly run the connection — the fetch is the capability check. A tool that can't reach the link can't reach GitHub either. Open the link yourself to read exactly what you're handing the AI.

Assistant won't fetch links? Paste the full instructions instead
full prompt 
Connect to my private GitHub repository <owner/repo> through the repo-bridge GitHub App using OAuth device flow.

Client ID: YOUR_CLIENT_ID

Steps:
1. Request a device code and user code from GitHub's device-flow endpoint for that Client ID.
2. Show me the user code and tell me to enter it at https://github.com/login/device, then wait while I authorize.
3. Exchange the authorization for a scoped, short-lived user-access token.
4. Confirm you can clone <owner/repo>, commit, and push back — and that access is limited to that repository only.

Never print the token.

ChatGPT opens in its default mode — switch to Agent mode to actually run it. Manus is a community-found shortcut.
